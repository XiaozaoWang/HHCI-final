Component 1: Overlapping Windows


A window is a rectangular area on the screen that displays the content of an application or file. Multiple windows can be open and arranged on the screen at the same time, allowing users to multitask and switch between different applications and files. Different types of windows include tiled windows, overlapping windows, cascading windows and so on.

Back in the era of Xerox and Apple, people are able to move, resize, scroll up and down, and organize the overlapping hierarchy of the windows. The significance of windows in the WIMP interface is that they allow users to multitask and work on multiple applications or files simultaneously. By using windows, users can arrange and view multiple applications or files on their screen at the same time, switch between them easily, and interact with them independently.

Windows also provide a visual representation of the content of an application or file, which makes it easier for users to understand and manipulate the content. For example, a document window displays the content of a text document in a way that is easy to read and edit. An image editing application can display an image in a window, allowing users to manipulate it using tools and options.
